# MUMU 음악 탐험대

## Context

초등 음악 수업에서 한 곡을 끝까지 들으며 가락의 흐름, 반복되는 프레이즈, 악기 음색을 손으로 확인하는 감상 활동이다. 정확한 입력 시각보다 떨어지는 노트 바와 실제 음악 사건의 연결을 우선한다.

## Target Users

- 기본 대상: 초등 4~6학년
- 기기: 교실 PC, 크롬북, 가로 방향 태블릿
- 활동 시간: 곡당 3~5분, 수업 내 2~3곡

## Learning Goal

학생은 일정한 박을 느끼고, 가락의 높낮이와 길이를 구별하며, 반복되는 부분과 주요 악기 음색을 찾아 말할 수 있다.

## Core Classroom Flow

1. 곡 카드에서 감상 주제를 확인한다.
2. 짧은 미리 듣기로 오늘의 감상 미션을 파악한다.
3. 5레인에서 떨어지는 노트 바를 직접 터뜨려 주요 가락을 연주한다.
4. 노트가 쉬는 집중 감상 구간에서 반주와 악기 변화를 듣는다.
5. 결과 화면에서 리듬 결과와 감상 질문을 따로 확인한다.

## Current Scope

- 서로 다른 감상 목표를 가진 빠른 클래식 4곡
- 마우스와 멀티 터치 기반 캔버스 직접 입력
- 넓은 가로 노트 바와 긴 노트 바
- 노트 수집률, 점수, 연속 팝
- 곡별 감상 미션과 결과 질문
- 교실 기기별 전체 음량 조절
- 최고 기록과 최근 감상 결과의 로컬 저장

## Non-goals

- 3D 렌더링과 물리 효과
- 경쟁 순위, 계정, 서버 저장
- 노래를 분석하지 않은 무작위 자동 채보
- 실패 시 곡 중단

## Data Model

- Song: 곡 정보, BPM, 박자, 주요 악기, 감상 미션
- MelodyNote: 시작 박, 길이, 음높이, 레인, 프레이즈
- Section: 도입/A/B/집중 감상/끝맺음 구간
- Result: 노트 수집률, 최고 연속 팝, 감상 질문 결과

## UI/UX Rules

- 낮은음은 왼쪽, 높은음은 오른쪽에 고정한다.
- 원근 투영 없이 고정형 2D 5레인을 사용하고 보이는 노트 바 전체를 터치 영역으로 삼는다.
- 게임을 시작하기 전 학생이 알아야 할 문장은 두 줄을 넘기지 않는다.
- 짧은 노트도 최소 24px 높이와 추가 터치 여백을 확보한다.
- 놓친 노트는 별도 질책 없이 화면 아래로 빠지고 곡은 계속된다.
- 배경은 감상 분위기를 만들되 노트보다 높은 대비를 갖지 않는다.
- 애니메이션 감소 설정을 존중한다.

## Technical Stack

- Vite + TypeScript
- Canvas 2D 렌더러
- Web Audio API 음악 시계와 무음정 입력 효과 재생
- FFmpeg 실제 연주 편집과 librosa 기반 온셋·음역 분석
- 선택한 곡의 완성 OGG만 우선 로드
- CSS 기반 화면 UI
- localStorage 기반 진행 기록

## Verification Plan

- TypeScript와 Vite 프로덕션 빌드 통과
- 곡 선택 → 미션 → 게임 → 질문 → 결과 전체 흐름 확인
- 1600x900 데스크톱, 1024x768 태블릿, 390x844 모바일 세로 화면 확인
- 60초 재생 후 음악과 노트의 체감 드리프트 확인
- 마우스와 멀티터치 직접 히트 테스트 확인
- 첫 화면과 현재 곡 외 에셋 지연 로딩 확인

## Risks

- 빠른 구간에서 여러 노트 바가 한 레인에 몰릴 가능성
- 생성 이미지 간 캐릭터·화풍 불일치
- 원녹음의 루바토와 강세 때문에 자동 분석 노트가 시각적으로 몰릴 가능성

## Next Steps

1. 네 곡을 실제 수업에서 플레이 테스트하고 노트 밀도를 조정한다.
2. 쉬움·보통·도전의 곡별 다중 차트를 추가한다.
3. 같은 제작 파이프라인으로 10곡 콘텐츠 팩을 확장한다.
