# MUMU 음악 탐험대

## Context

초등 음악 수업에서 한 곡을 끝까지 들으며 박, 가락의 흐름, 반복되는 프레이즈, 악기 음색을 몸으로 확인하는 감상 활동이다. 노트 수를 늘리는 순발력 게임보다 실제 음악 사건과 학생 입력의 연결을 우선한다.

## Target Users

- 기본 대상: 초등 3~4학년
- 확장 대상: 1~2학년 3레인 모드, 5~6학년 심화 채보
- 기기: 교실 PC, 크롬북, 가로 방향 태블릿
- 활동 시간: 곡당 3~5분, 수업 내 2~3곡

## Learning Goal

학생은 일정한 박을 느끼고, 가락의 높낮이와 길이를 구별하며, 반복되는 부분과 주요 악기 음색을 찾아 말할 수 있다.

## Core Classroom Flow

1. 곡 카드에서 감상 주제를 확인한다.
2. 짧은 미리 듣기로 오늘의 감상 미션을 파악한다.
3. 5레인 리듬 액션으로 주요 가락을 연주한다.
4. 노트가 쉬는 집중 감상 구간에서 반주와 악기 변화를 듣는다.
5. 결과 화면에서 리듬 결과와 감상 질문을 따로 확인한다.

## MVP Scope

- 서로 다른 감상 목표를 가진 3곡
- 키보드 A/S/D/J/K와 5개 터치 패드
- 탭 노트와 긴 음표
- Perfect/Great/Good/Miss 판정, 점수, 콤보
- 곡별 감상 미션과 결과 질문
- 기기별 오디오 지연 조절
- 최고 기록과 최근 감상 결과의 로컬 저장

## Non-goals

- 3D 렌더링과 물리 효과
- 경쟁 순위, 계정, 서버 저장
- 노래를 분석하지 않은 무작위 자동 채보
- 실패 시 곡 중단

## Data Model

- Song: 곡 정보, BPM, 박자, 음색 세계, 감상 미션
- MelodyNote: 시작 박, 길이, 음높이, 레인, 프레이즈
- Section: 도입/A/B/집중 감상/끝맺음 구간
- Harmony: 마디별 화음 진행
- Result: 정확도, 최고 콤보, 감상 질문 결과

## UI/UX Rules

- 낮은음은 왼쪽, 높은음은 오른쪽에 고정한다.
- 원근 투영 없이 고정형 2D 5레인을 사용하고 노트 중심과 터치 패드 중심을 항상 일치시킨다.
- 게임을 시작하기 전 학생이 알아야 할 문장은 두 줄을 넘기지 않는다.
- 터치 패드는 최소 64px 높이를 확보한다.
- MISS는 질책하지 않고 다음 박으로 바로 돌아오게 한다.
- 배경은 감상 분위기를 만들되 노트보다 높은 대비를 갖지 않는다.
- 애니메이션 감소 설정을 존중한다.

## Technical Stack

- Vite + TypeScript
- Canvas 2D 렌더러
- Web Audio API 음악·판정 시계
- 피아노·현악·관악·금속 타악기 샘플을 곡 데이터에서 함께 스케줄링하는 다층 반주
- CSS 기반 화면 UI
- localStorage 기반 진행 기록

## Verification Plan

- TypeScript와 Vite 프로덕션 빌드 통과
- 곡 선택 → 미션 → 게임 → 질문 → 결과 전체 흐름 확인
- 1440x900 데스크톱, 1024x768 태블릿, 844x390 모바일 가로 화면 확인
- 60초 재생 후 음악과 노트의 체감 드리프트 확인
- 키보드와 멀티터치 입력 확인
- 첫 화면과 현재 곡 외 에셋 지연 로딩 확인

## Risks

- 브라우저와 블루투스 기기별 오디오 지연 차이
- 생성 이미지 간 캐릭터·화풍 불일치
- 합성 악기 음색이 장시간 반복될 때의 피로감

## Next Steps

1. 반짝반짝 작은 별 완성형으로 입력과 음악 싱크를 검증한다.
2. 환희의 노래와 달빛 왈츠를 추가해 4박·3박 감상을 비교한다.
3. 수업 테스트 후 10곡 콘텐츠 팩으로 확장한다.
